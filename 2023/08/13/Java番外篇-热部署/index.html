热部署(番外篇)

Created2023-08-13|Updated2023-08-17|Java
|Post View:

1.什么是热部署

是指在不关闭或重启服务的情况下，更新Java类文件或配置文件，实现修改内容生效；通过热部署，可提高开发效率，节省程序打包重启的时间，同时，可实现生产环境中需要不停机或重启的服务的升级。

2.热部署是如何实现的

在不使用热部署机制时，当项目代码发生修改时，往往需要重启应用来查看修改效果。其本质是重新编译生成新的Class文件，然后Class文件被虚拟机的ClassLoader加载。

在使用热部署机制时，当坚挺到有Class文件发生改动，就会创建一个新的ClassLoader加载改动文件。其本质是使用了两个ClassLoader，一个用于加载那些不会改变的类(第三方Jar包)，另一个ClassLoader加载会更改的类，称为restart ClassLoader。当代码再次发生修改时，原先的restart ClassLoader被丢弃，重新创建新的restart ClassLoader。由于restart ClassLoader需要加载的类较少，重启时间便大大缩短。

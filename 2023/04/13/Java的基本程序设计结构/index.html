Java的基本程序设计结构

Created2023-04-13|Updated2023-08-17|Java
|Post View:

1. 类与源代码的命名规范

类名必须以大写字母开头，后面可以跟字母和数字的任意组合。源代码的文件名必须与公共类(public)的名字相同，并用.java作为扩展名。

2. main方法

运行已编译程序时，Java虚拟机总是从指定类中的main方法开始执行，在类的源文件中必须包含一个main方法。根据Java语言规范，main方法必须声明为public。Java中的main方法必须是静态的。

3. 强类型

Java是一种强类型语言，必须为每一个变量声明一种类型。

4. 数据类型

Java提供4种整型。在Java中，整型的范围与运行Java代码的机器无关。

Java提供2种浮点类型。float类型的数值后有一个后缀F或f(例如，3.14F)，没有后缀的浮点数值总默认为double。

char类型的字面量值要用单引号括起来。双引号括起来的是字符串。以下是转义字符表。

boolean类型有两个值：false和true。整型值和布尔值之间不能进行相互转换。

5. 变量

声明一个变量之后，必须用赋值语句对变量进行显式初始化，不能使用未初始化的变量的值。

从Java 10开始，对于局部变量，如果可以从变量的初始值推断出它的类型，就不再需要声明类型，只需要使用关键字var。

6. 常量

在Java中，利用关键字final表示常量，常量只能被复制一次，一旦被赋值就不能够再更改，习惯上常量名使用全大写。

在Java中，经常希望某个常量在一个类的多个方法中使用，这些常量称为类常量，使用static final设置一个类常量。

7. 枚举类型

枚举类型包括有限个命名的值，例如：

enum Size {SMALL， MEDIUM， LARGE， EXTRA_LARGE}

11. 字符串

Java文档中将String类对象称为是不可变的，修改字符串的操作实际上是新开辟了内存，让引用变量指向了存有新字符串的内存空间。

Java编译器可以让字符串共享，各种字符串存放在公共的存储池中，字符串变量指向存储池中相应的位置，如果复制一个字符串变量，原始字符串与复制字符串共享相同的字符。值得注意的是，Java虚拟机并不是始终将相同的字符串共享，实际上只有字符串字面量是共享的，而+或substring等操作得到的字符串并不共享。

可以使用equals方法检测两个字符串是否相等，不要使用==运算符检测字符串是否相等，==运算符实际上比较的是两个引用变量是否指向相同的内存空间。由于并不是所有的相同字符串都触发了字符串共享机制，==判断是有风险的。

由于每次拼接字符串时，都会创建一个新的String对象，耗时又浪费空间，使用StringBuilder类可以避免上述问题。

StringBuilder builder=new StringBUilder();
builder.append(ch)
builder.append(str)

12. while与do/while

while会先执行判断条件，所以while循环体中的代码可能一次都不执行。

do/while会在执行循环体后再执行判断条件，do/while则至少执行一次循环体。

13. switch语句

注意，如果在case分支语句的末尾没有break语句，那么就会接着执行下一个case分支语句。

14. 数组

数组一旦创建，就不能改变长度，如果需要扩展数组大小，则使用另一种数据结构-数组列表(array list)。

15. 命令行参数

每一个Java应用程序都有一个带String arg[]参数的main方法，该参数表明main方法将接收一个字符串数组，也就是命令行上指定的参数。

16. 数组排序

要对数值型数组排序，可以使用Array类中的sort方法，该方法使用了优化的快速排序算法。

int[] a=new int [10000];

Arrays.sort(a);
